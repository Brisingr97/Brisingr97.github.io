How not to : ORB SLAM

Oct 1 2018

My experiments with trying to run ORB SLAM on a VM and then a RPi Zero M

Enroute :
I cloned the ORB SLAM repository from here and started cloning into the required dependecies starting from ROS
However the first hiccup was that ORB SLAM (1) was tested only on ROS which were compatible with Ubuntu 12.04 and I was running 16.04 on my VM.

Onto Ubuntu 14.04 with ROS indigo! Hiwever there was another issue, the version of OpenCV was a mismatch, So a fresh installation of OpenCV (v2.4) was required, which surpringly took ~11 Gigs of space almost depeleting my puny 20GB VM harddrive.
New VM ,60Gigs HDD, Repeat

After all the version aligned and compling the code, There was an issue,

" typedef Eigen::Map, Matrix::Flags & AlignedBit ? Aligned : Unaligned > HessianBlockType;"

Which meant that the version of my Eigen library was wrong. Their releases can be found here

After spending almost 2 days on Stack-Overflow looking for specific compilation error, I learnt that because of such issues ORB SLAM was refreshed to ORB SLAM 2. Which could work on Ubuntu 16.04.
Back to old VM.
Downloaded Pangolin
(installs like butter) Downloaded Eigen(please use 3.1 and not 3.2 otherwise you'll get the same problem)
ROS was not required.
I ran it on TUM and KITTI datasets and then moved the system to a more mobile version to test the efficiency.
